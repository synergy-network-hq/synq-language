# Contributing to SynQ

Thank you for your interest in contributing to SynQ.

SynQ is an open-source project with stewarded standards. Contributions are welcome, but must align with the project’s security, determinism, and governance principles.

---

## 1. Contribution Scope

We welcome contributions in:

- Compiler and VM improvements
- Tooling and SDKs
- Documentation and specifications
- Testing and examples
- Security research and analysis

---

## 2. Contribution Requirements

All contributors MUST:

- Agree to the SynQ Contributor License Agreement (CLA)
- Ensure contributions are original or properly licensed
- Avoid introducing non-deterministic behavior
- Follow documented coding and review standards

---

## 3. Review Process

All contributions undergo:

- Automated testing
- Manual code review
- Security and determinism assessment
- Governance review for protocol-affecting changes

Acceptance is at the discretion of SynQ maintainers.

---

## 4. Protocol-Level Changes

Changes affecting:

- Language semantics
- VM execution rules
- Cryptographic primitives
- Gas or determinism guarantees

Require:

- Formal proposal
- Public review period
- Explicit governance approval

---

## 5. Security Issues

Security vulnerabilities MUST be reported privately following the security disclosure policy.

Do NOT open public issues for undisclosed vulnerabilities.

---

## 6. Code of Conduct

All contributors are expected to adhere to the project’s Code of Conduct.

---

## 7. No Rights Granted

Contributing does not grant:

- Trademark rights
- Certification authority
- Governance authority

These remain stewarded to protect ecosystem integrity.
